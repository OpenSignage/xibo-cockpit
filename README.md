# Xibo Cockpit

Xibo Cockpitは、Xibo Agent用のインタラクティブチャットクライアントアプリケーションです。直感的なインターフェースを通じてXibo Agentとの対話を可能にし、様々なコマンドや質問に対するレスポンスを取得できます。

## 機能

- AIエージェントとのチャット対話
- 会話履歴の保存と管理
- マークダウン対応のレスポンス表示
- コードブロックのシンタックスハイライト
- ダークモード/ライトモードの切り替え
- 設定のカスタマイズと保存
- オフラインモードでの動作

## アプリケーション構造

│   ├── index.html            - メインアプリケーションのHTMLドキュメント

│   └── js/                   - JavaScriptモジュール

- 会話履歴の管理
- IME入力処理（日本語入力対応）

このファイルにはさまざまなJavaScript関数が含まれており、主に次の目的で使用されます：
- エージェントサーバーとの接続確立

- 会話履歴の保存と表示
- 設定のロード/適用

- メッセージバブルのスタイル

マークダウンテキストの処理と表示を担当するモジュールです。主な機能：

メッセージの表示と処理を担当するモジュールです。主な機能：

UIの制御と対話を管理するモジュールです。主な機能：

## 主要機能の解説

### 会話履歴管理システム

アプリケーションはブラウザのローカルストレージを使用して会話履歴を保存します：

- 各会話はユニークIDで識別
- 会話のタイトルは最初のユーザーメッセージから自動生成
- 会話履歴はサイドバーに一覧表示
- 会話の切り替え・削除が可能
- 最大保存数は設定から変更可能

### マークダウン対応

- マークダウン構文のパース
- コードブロックのシンタックスハイライト
- テーブル表示のサポート
- リストとインデントの適切な表示
- Mermaidダイアグラム描画機能

### 設定管理

- サーバー接続URLの設定
- 表示モード（ダーク/ライト）の切り替え
- 言語設定（日本語/英語）
- 会話履歴の最大保存数設定
- 設定はローカルストレージに保存

### IME入力対応

- 日本語・中国語などの文字入力に対応
- IME確定時の誤送信防止
- 入力フィールドの自動リサイズ

### エラーハンドリング

- 接続エラー時のフォールバック
- オフラインモードのサポート
- エラーメッセージの表示

## 使い方

### ローカルで実行する

アプリケーションは静的HTMLファイルであるため、単純にブラウザで開くだけで実行できます：

1. `dist/index.html` ファイルをお好みのWebブラウザで開きます
2. またはローカルのHTTPサーバーを使って実行：
   ```bash
   # Node.jsのhttp-serverを使用する場合
   npx http-server dist
   
   # Pythonの組み込みHTTPサーバーを使用する場合
   cd dist && python -m http.server 8080
   ```
   その後、ブラウザで `http://localhost:8080` にアクセスします

### Xiboサーバーに接続する

1. 設定ボタン（歯車アイコン）をクリックして設定モーダルを開きます
2. 「接続先URL」にXiboエージェントサーバーのURLを入力します（デフォルト: `http://localhost:4111`）
3. 「保存」ボタンをクリックして設定を適用します
4. サーバーに接続し、正常に接続されたかメッセージが表示されます

### 基本的な使い方

1. メッセージエリアに質問や指示を入力します
2. 送信ボタンまたはEnterキーでメッセージを送信します
3. AIアシスタントからの応答が表示されます
4. 左側のサイドバーで過去の会話を選択したり、新しい会話を開始できます
5. 設定ボタンから表示モードや言語、サーバーURLを変更できます

### 会話履歴の管理

- **新規会話**: 左側の「新規会話」ボタンをクリックすると、新しい会話が開始されます
- **会話履歴表示**: 左側のサイドバーに過去の会話一覧が表示されます
- **会話切り替え**: 履歴の項目をクリックすると、その会話に切り替わります
- **会話削除**: 各会話項目にマウスを置くと表示される削除ボタンで個別の会話を削除できます
- **すべての会話を削除**: 会話履歴ヘッダーのゴミ箱アイコンですべての履歴を削除できます

### マークダウンの使用

AIアシスタントの応答はマークダウン形式で表示されます：

- コードブロックは構文ハイライトされて表示されます
- テーブルは整形された表として表示されます
- リストやヘッダーなどのマークダウン要素が適切に表示されます
- 数式の表示にも対応しています
- Mermaidダイアグラム構文をサポートしています

## 開発とカスタマイズ

### 依存ライブラリ

- markdown-it: マークダウンパーサー
- highlight.js: コードシンタックスハイライト
- mermaid: ダイアグラム描画ライブラリ

### 開発環境のセットアップ

```bash
# リポジトリのクローン
git clone [リポジトリURL]

# プロジェクトディレクトリに移動
cd xibo-cockpit

# 依存関係のインストール
npm install

# 開発サーバーの起動
npm run dev
```

### ビルド手順

```bash
# 本番用ビルド
npm run build
```

ビルド後のファイルは `dist` ディレクトリに出力されます。

### デバッグ方法

1. ブラウザの開発者ツールを開きます（F12またはCtrl+Shift+IまたはCmd+Option+I）
2. コンソールタブでエラーメッセージを確認できます
3. 接続やメッセージの問題はコンソールログを確認してください
4. ネットワークタブでAPIリクエストの状態を監視できます

### カスタマイズのヒント

- **スタイルの変更**: `dist/css/styles.css` を編集してUIの外観をカスタマイズできます
- **サーバー接続設定の変更**: `dist/js/config.js` のデフォルト設定を編集できます
- **マークダウン処理のカスタマイズ**: `dist/js/markdown.js` を編集して、マークダウン表示をカスタマイズできます

## 注意点

- ローカルストレージを使用しているため、ブラウザのプライベートモードや履歴削除機能を使用すると会話履歴が失われる可能性があります
- 本番環境にデプロイする場合は、セキュリティ面を考慮してAPIエンドポイントの設定を見直してください
- アプリケーションはブラウザのローカルストレージに依存しているため、大量のデータを扱う場合はパフォーマンスに影響する可能性があります

## トラブルシューティング

#### サーバーに接続できない場合
- 設定で正しいサーバーURLが入力されているか確認してください
- サーバーが起動していて、ネットワークアクセス可能な状態か確認してください
- ブラウザのコンソールでネットワークエラーを確認してください

#### マークダウンが正常に表示されない場合
- すべての必要なライブラリが読み込まれているか確認してください
- ブラウザのJavaScriptが有効になっているか確認してください
- コンソールでエラーメッセージを確認してください

#### IME入力で問題が発生する場合
- 最新バージョンのブラウザを使用しているか確認してください
- 入力中に誤って送信ボタンを押さないようにしてください
